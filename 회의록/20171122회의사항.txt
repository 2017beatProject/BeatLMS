2017-11-22 회의사항

1. 데이터 베이스 구성

-큰 묶음: 회원정보, 강의관리(출석/성적), 권한관리, 공지사항 관리, 상담관리
- 회원 테이블 구성은  지위식별코드(가령 직원 1, 강사 2, 수강생 3)와 회원번호 두가지로 구분
: 이 테이블을 PK는 회원가입시 부여되는 회원번호

- 직원, 수강생 모두 하나의 회원 테이블 하나로 관리

-수강생에 대해 별도로 출석, 성적 정보를 입력해야 하므로 수강생테이블생성해서 회원테이블의 회원식별번호와 회원구분


2. 출석 관리는 결석, 조퇴, 가계에 대한 테이블을 따로 따고(결석일, 조퇴일(조퇴시간), 지각일(지각시간) ) 출석은 수강생 테이블에서 일괄적으로 카운팅

3. 목요일 pt는 유스케이스 필수 지참(가능하면 출력해올것...)
시나리오는 선택사항, 데이터모델이나 메뉴 모델 선택사항
*호경씨는 메뉴 디자인 대략적으로 챙겨와주셔서 고맙스빈다

4. 일정
-스토리보드 팀(메뉴구성 포함)
-ER-D 팀
 각 설계와 코딩이 동시진행 예정.
- 클래스 구성과 시퀀스는 두 팀 내용을 보고 협업 

*호경-css, 요한-java script, 태경-control, 민건-model 및 오라클 쿼리문
*희정씨의 경우 전담 파트는 JAVA script로 하고, DB를 서브로 진행, 스토리보드 팀에서 메뉴모델링 초안 담당
* 이상의 내용은 pt 이후 변동 가능.

6. 각 파트 시작지점
- DB: DTO, DAO 분류를 어떻게 할것인지 
- Controll: 각 테이블별 패키지 구성을 어떻게 할지
--Front- 
-CSS: 그리드 구성, 메뉴 구성, 스토리보드 구성
-js: 페이지별 정보전달 방안 구상(세션, 리퀘스트, 어트리뷰트), 필요 기능 설정( bxslider 등 사용 라이브러리 탐색)
